Section A.4 Units

Pretext doesn't have great support for units, but they are very important for engineering, so I have cobbled together a workaround. This isn't valid pretext, so it won't validate with the standard schema, but you can use weh-pretext.rncm in the custom folder to get it to validate.

It it will be expanded to valid pretext syntax before the output is made.

weh

Whenever you have a quantity with units, you need to include it in the source code like this, and put this in a math environment like <m>, <me>, or <md>.

<qty units='N'>

The values inside the tag can be a bare number, or any valid math.

By doing this, when the book is turned into html or pdf, the units will be rendered correctly, Upright font in a math environment, superscripts etc.

See samples below.

These abbreviations get translated by my units.xsl which is called as part of scriv2ptx.xsl

  1. \(\displaystyle 10~\text{ lb} \)
  2. \(\displaystyle 100~\text{ N} \)
  3. \(\displaystyle 1000~\text{ kN} \)
  4. \(\displaystyle 10~\text{ MN} \)
  5. \(\displaystyle 1.1~\text{ m} \)
  6. \(\displaystyle 1.1~\text{ cm} \)
  7. \(\displaystyle 1.1~\text{ mm} \)
  8. \(\displaystyle 1.1~\text{ ft} \)
  9. \(\displaystyle 1.1~\text{ in} \)
  10. \(\displaystyle 12~\text{ N-m} \)
  11. \(\displaystyle 12~\text{ ft-lb} \)
  12. \(\displaystyle 12~\text{ in-lb} \)
  13. \(\displaystyle 12~\text{ lb/ft} \)
  14. \(\displaystyle 12~\text{ N/m} \)

If you need a unit which is not found here, add it to the xslt in custom/stage-1.xslt
