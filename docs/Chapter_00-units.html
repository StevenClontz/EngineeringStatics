Section A.5 Units

Pretext doesn't have great support for units, but they are very important for engineering, so I have cobbled together a workaround. This isn't valid pretext, so it won't validate with the standard schema, but you can use weh-pretext.rnc in the custom folder to get it to validate.

It it will be expanded to valid pretext syntax before the output is made.

Whenever you have a quantity with units, you need to include it in the source code like this, and put this in a math environment like <m>, <me>, or <md>.

<qty units='N'>

The values inside the tag can be a bare number, or any valid math.

By doing this, when the book is turned into html or pdf, the units will be rendered correctly, Upright font in a math environment, superscripts etc.

See samples below.

If you need a unit which is not here, add it a new macro to the doc info
